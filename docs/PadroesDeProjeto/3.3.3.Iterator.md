## 1. Introdução

O **Iterator** é um padrão de projeto comportamental catalogado pela Gang of Four (GoF) que fornece uma maneira de acessar sequencialmente os elementos de uma coleção sem expor sua representação interna (GAMMA et al., 1995).

Em outras palavras, o padrão Iterator separa o **algoritmo de navegação** (percorrer, filtrar, pular) da **estrutura de dados** que armazena os elementos. O cliente (o código que consome a coleção) interage apenas com a interface do Iterador, sem precisar saber se a coleção é um Array, uma Árvore, um Grafo ou qualquer outra estrutura.

### Quando aplicar?

- Quando se deseja percorrer uma coleção sem expor seus detalhes internos de implementação;
- Quando se precisa de **múltiplas formas de navegar** pela mesma coleção (ex.: linear, filtrada, reversa);
- Quando se quer desacoplar os algoritmos de traversal da estrutura da coleção, respeitando o Princípio da Responsabilidade Única.

## 2. Metodologia

Para a implementação deste padrão, a equipe partiu da análise das User Stories relacionadas à navegação de conteúdo no [Product Backlog](https://unbarqdsw2026-1-turma01.github.io/2026.1-T01-_G2_ConhecendoRequisitos_Entrega_02/#/Modelagem/2.5.3.Backlog) da [Entrega 2](https://unbarqdsw2026-1-turma01.github.io/2026.1-T01-_G2_ConhecendoRequisitos_Entrega_02/#/), buscando identificar onde o padrão se encaixaria de forma genuína no domínio do projeto.

A aplicação foi desenvolvida sobre uma **adaptação simplificada** da estrutura de Trilhas de Aprendizado da plataforma. A fim de demonstrar a aplicação do padrão de forma mais clara e didática, optamos por abstrair a complexidade real do domínio (que envolveria a hierarquia completa de módulos, sub-módulos, perguntas e alternativas de um quiz) e modelamos a `Trilha` como uma coleção direta de objetos básicos do tipo `Conteudo` (`"texto"`, `"video"` e `"quiz"`). A partir dessa abstração, a equipe optou por implementar dois iteradores concretos, cada um correspondendo a um fluxo real de uso do sistema:

- **`IteradorLinear`** — percorre todos os conteúdos em ordem sequencial, refletindo o fluxo natural de estudo diário de um Aluno;
- **`IteradorRevisao`** — atua como um filtro, retornando exclusivamente os conteúdos do tipo `"quiz"`, atendendo ao cenário em que o Aluno deseja focar na resolução de exercícios antes de uma avaliação.

A implementação foi feita em **JavaScript (Node.js)**, linguagem de maior familiaridade da equipe conforme levantado na [Entrega 1](https://unbarqdsw2026-1-turma01.github.io/Grupo02_ConhecendoRequisitos_Entrega01/#/Base). O código produzido atua como uma Prova de Conceito (PoC), priorizando evidenciar as vantagens de desacoplar o algoritmo de navegação da estrutura de dados sobre a integração com a complexidade arquitetural completa do projeto.

## 4. Descrição das Classes

### 4.1. `Conteudo` (Element)

Representa um item individual de uma Trilha. É o objeto que os Iteradores retornam a cada chamada de `proximo()`.

| Atributo / Método        | Descrição                                                         |
| ------------------------ | ----------------------------------------------------------------- |
| `titulo: String`         | Nome do conteúdo (ex.: "Introdução à Elicitação").                |
| `tipo: String`           | Categoria do conteúdo: `"video"`, `"texto"` ou `"quiz"`.          |
| `exibirConteudo(): String` | Retorna uma representação formatada para exibição no terminal.    |

### 4.2. `Trilha` (ConcreteAggregate)

A coleção principal. Além de armazenar os conteúdos, é responsável por fabricar (instanciar) os iteradores para cada modo de uso.

| Atributo / Método                   | Descrição                                                  |
| ----------------------------------- | ---------------------------------------------------------- |
| `nome: String`                      | Nome da Trilha de Aprendizado.                             |
| `_conteudos: List~Conteudo~`        | Lista interna de conteúdos (encapsulada).                  |
| `adicionarConteudo(c: Conteudo)`    | Adiciona um item à Trilha.                                 |
| `criarIteradorLinear(): IIterador`  | Fabrica um `IteradorLinear` para percorrer todos os itens. |
| `criarIteradorRevisao(): IIterador` | Fabrica um `IteradorRevisao` para percorrer só os quizzes. |

### 4.3. `IteradorLinear` (ConcreteIterator)

Percorre todos os conteúdos da Trilha, do índice `0` ao último, sem nenhum filtro.

| Atributo / Método       | Descrição                                                          |
| ----------------------- | ------------------------------------------------------------------ |
| `_conteudos: Array`     | Referência à lista interna da Trilha.                              |
| `_posicao: int`         | Cursor que rastreia o elemento atual na iteração.                  |
| `temProximo(): bool`    | Retorna `true` enquanto existirem elementos não visitados.         |
| `proximo(): Conteudo`   | Retorna o próximo `Conteudo` e avança o cursor.                    |

### 4.4. `IteradorRevisao` (ConcreteIterator)

Aplica um filtro ao construir sua lista interna, mantendo apenas os conteúdos com `tipo === "quiz"`. O cliente não precisa saber que existe um filtro; ele usa a mesma interface do `IteradorLinear`.

| Atributo / Método       | Descrição                                                                        |
| ----------------------- | -------------------------------------------------------------------------------- |
| `_quizzes: Array`       | Sublista filtrada, contendo apenas os conteúdos do tipo `"quiz"`.                |
| `_posicao: int`         | Cursor que rastreia o quiz atual.                                                |
| `temProximo(): bool`    | Retorna `true` enquanto houver quizzes não visitados.                            |
| `proximo(): Conteudo`   | Retorna o próximo quiz e avança o cursor.                                        |

## 5. Implementação

O código a seguir implementa o padrão Iterator em **JavaScript (Node.js)**, seguindo fielmente o diagrama de classes do projeto. O JavaScript foi escolhido por ser a linguagem com maior familiaridade da equipe, conforme observado na aba [Conhecimentos da Equipe](https://unbarqdsw2026-1-turma01.github.io/Grupo02_ConhecendoRequisitos_Entrega01/#/Base/1.5.4.Conhecimentos) realizado na [Entrega 1](https://unbarqdsw2026-1-turma01.github.io/Grupo02_ConhecendoRequisitos_Entrega01/#/Base).

```javascript
// PRODUTO (Conteúdo da Trilha)

/**
 * Representa um item de conteúdo dentro de uma Trilha.
 * Pode ser do tipo "video", "texto".
 */
class Conteudo {
  constructor(titulo, tipo) {
    this.titulo = titulo;
    this.tipo = tipo; // "video" | "texto" 
  }

  toString() {
    return `[${this.tipo.toUpperCase()}] ${this.titulo}`;
  }
}

/**
 * Itera sobre todos os conteúdos da trilha na ordem original.
 * Usado no fluxo de estudo diário do Aluno.
 */
class IteradorLinear {
  constructor(conteudos) {
    this._conteudos = conteudos;
    this._posicao = 0;
  }

  temProximo() {
    return this._posicao < this._conteudos.length;
  }

  proximo() {
    if (!this.temProximo()) throw new Error("Não há mais conteúdos.");
    return this._conteudos[this._posicao++];
  }
}

/**
 * Itera apenas sobre os conteúdos do tipo "quiz".
 * Usado pelo Aluno em modo de revisão.
 */
class IteradorRevisao {
  constructor(conteudos) {
    this._quizzes = conteudos.filter((c) => c.tipo === "quiz");
    this._posicao = 0;
  }

  temProximo() {
    return this._posicao < this._quizzes.length;
  }

  proximo() {
    if (!this.temProximo()) throw new Error("Não há mais quizzes para revisão.");
    return this._quizzes[this._posicao++];
  }
}

/**
 * Representa uma Trilha de aprendizado.
 * Funciona como a Coleção que fabrica seus próprios Iteradores.
 */
class Trilha {
  constructor(nome) {
    this.nome = nome;
    this._conteudos = [];
  }

  adicionarConteudo(conteudo) {
    this._conteudos.push(conteudo);
  }

  // Fábrica de Iterador para estudo linear (dia a dia)
  criarIteradorLinear() {
    return new IteradorLinear(this._conteudos);
  }

  // Fábrica de Iterador para modo revisão (apenas quizzes)
  criarIteradorRevisao() {
    return new IteradorRevisao(this._conteudos);
  }
}

// CLIENTE (Simulando o Aluno na Plataforma)


// Montando a Trilha de Engenharia de Requisitos
const trilha = new Trilha("Engenharia de Requisitos");
trilha.adicionarConteudo(new Conteudo("O que são Requisitos?", "texto"));
trilha.adicionarConteudo(new Conteudo("Introdução à Elicitação", "video"));
trilha.adicionarConteudo(new Conteudo("Quiz: Conceitos Básicos", "quiz"));
trilha.adicionarConteudo(new Conteudo("Técnicas de Entrevista", "texto"));
trilha.adicionarConteudo(new Conteudo("Casos de Uso na Prática", "video"));
trilha.adicionarConteudo(new Conteudo("Quiz: Técnicas de Elicitação", "quiz"));

// ── Cenário 1: Aluno no estudo diário (percorre tudo na ordem)
console.log(`\n=== Modo Estudo: Trilha "${trilha.nome}" ===`);
const iteradorLinear = trilha.criarIteradorLinear();
while (iteradorLinear.temProximo()) {
  console.log("  →", iteradorLinear.proximo().toString());
}

// ── Cenário 2: Aluno em modo revisão (percorre apenas os Quizzes)
console.log(`\n=== Modo Revisão: Apenas Quizzes da Trilha "${trilha.nome}" ===`);
const iteradorRevisao = trilha.criarIteradorRevisao();
while (iteradorRevisao.temProximo()) {
  console.log("  →", iteradorRevisao.proximo().toString());
}
```

## 8. Senso Crítico

Quando começamos a estudar o padrão Iterator, a pergunta central da equipe foi: onde no nosso projeto precisamos realmente percorrer uma coleção de formas distintas? A resposta surgiu naturalmente quando pensamos na experiência do Aluno navegando por uma Trilha de Aprendizado.

Uma Trilha no "ConhecendoRequisitos" não é uma lista estática qualquer. Dependendo do contexto, o aluno interage com ela de maneiras completamente diferentes. No estudo diário, ele avança pelo conteúdo em ordem, passando por textos, vídeos e quizzes conforme o Administrador organizou. Já na véspera de uma avaliação, por exemplo, ele não quer revisitar os vídeos em ordem; ele quer ir direto nos conteudos que serão cobrados na avaliação. São dois comportamentos distintos sobre a mesma estrutura de dados, e o Iterator resolve isso de forma muito elegante.

Sem o padrão, essa lógica de filtragem provavelmente acabaria espalhada por controladores e pelo Front-end, com laços e condicionais duplicados em múltiplos arquivos. Cada novo modo de navegação que surgisse, como "mostrar apenas conteúdos ainda não concluídos", exigiria buscar e replicar esse código em vários pontos do sistema. Com o Iterator, a própria `Trilha` assume a responsabilidade de fabricar seus modos de navegação. O cliente simplesmente escolhe o iterador adequado e usa sempre a mesma interface (`temProximo()`, `proximo()`), sem precisar conhecer os detalhes internos da coleção.

Um questionamento legítimo que surgiu durante as discussões foi: *"o JavaScript já oferece `for...of` e métodos nativos como `.filter()`, então por que formalizar um Iterator?"*. Para casos simples, esses recursos são suficientes. No entanto, o padrão Iterator cria um contrato explícito e estável que desacopla a lógica de navegação de quem consome a coleção. Se a estrutura interna da Trilha evoluir de um Array simples para algo mais complexo, como uma árvore de módulos e submódulos, os Iteradores se adaptam internamente e o restante do sistema permanece intocado. Essa resiliência é o que torna o padrão valioso em projetos com tendência de crescimento.

### Comentários sobre o Trabalho em Equipe

A implementação do Iterator foi um exercício coletivo valioso para consolidar os conceitos de encapsulamento e separação de responsabilidades que permearam toda a Entrega 3. O ponto de virada para o grupo foi perceber que o problema não era "como fazer um laço", mas sim "como separar o algoritmo de navegação da estrutura de dados". A partir dessa compreensão, o padrão se encaixou naturalmente. Foi também muito legal ver como o Iterator dialoga com os padrões criacionais já entregues: enquanto o Builder e o Multiton tratam da criação e do acesso aos dados, o Iterator define como esses dados são consumidos e apresentados ao Aluno ao longo de sua jornada de aprendizado.
