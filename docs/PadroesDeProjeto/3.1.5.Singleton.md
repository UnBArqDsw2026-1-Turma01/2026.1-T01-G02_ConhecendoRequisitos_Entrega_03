## Introdução

O presente documento tem como objetivo apresentar a implementação do padrão de projeto Singleton aplicado à classe XXX, desenvolvida no contexto do nosso projeto acadêmico.
O padrão Singleton é um dos padrões criacionais do catálogo GoF (Gang of Four) e tem como finalidade garantir que uma classe possua apenas uma instância em todo o sistema, fornecendo um ponto global de acesso a ela (Refactoring Guru, 2025).

## Motivação

A motivação principal para o uso do Singleton na classe XXX é garantir o controle centralizado das publicações em um único ponto do sistema. Com isso, evita-se a criação de múltiplas instâncias que poderiam ocasionar inconsistências de dados, como listas de publicações divergentes entre diferentes partes do código (Refactoring Guru, 2024).
